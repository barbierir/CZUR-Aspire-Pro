# Release v0.1.0 — First Public Linux Release

Book Capture `v0.1.0` is the first public release focused on practical Linux document scanning workflows for unsupported scanner hardware exposed as V4L2/UVC cameras.

## Highlights

- First stable public release candidate for Linux users.
- Desktop workflow for capture, session management, and PDF export.
- Multiple distribution options: portable AppImage, `.deb`, and user-local installer.
- Linux desktop integration through launcher, desktop entry, and icon support.

## Install options

Choose the option that best fits your setup:

- **AppImage (`book-capture-x86_64.AppImage`)**: best for portable use without system-wide installation.
- **Debian package (`book-capture_0.1.0_amd64.deb`)**: best for Ubuntu/Zorin users who want package-based install/remove.
- **User-local installer (`install.sh`)**: best when you want a local install under `~/.local` directly from the repository.

## Supported use cases

- Book/page capture using V4L2-compatible cameras (`/dev/video*`).
- Session-based scanning and page organization.
- Basic image cleanup before PDF export.
- Linux users with hardware lacking official vendor software.

## Known limitations

- OCR and searchable PDFs are not included in `v0.1.0`.
- Final quality depends on camera placement, lighting, and page stability.
- Some page flattening/perspective scenarios may still require manual adjustments.
- This project does not provide hardware drivers; the camera must already work in Linux.

## Checksums (recommended)

Generate checksums from the folder containing release artifacts:

```bash
sha256sum book-capture-x86_64.AppImage book-capture_0.1.0_amd64.deb > SHA256SUMS
```

Users can verify downloads with:

```bash
sha256sum -c SHA256SUMS
```
